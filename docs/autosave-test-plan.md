# 自动保存功能测试计划

## 测试设备
- 设备: Supernote Nomad (SN078C10000377)
- 应用: NomadMark-debug-1.0.0.apk

## 测试场景

### 测试 1: 自动保存基础功能
**目标**: 验证编辑后自动保存到临时文件

**步骤**:
1. 打开应用，创建新文档
2. 输入测试内容: "AutoSave Test Content"
3. 等待 3 秒（防抖延迟）
4. 通过 adb 检查临时文件是否存在

**预期结果**:
- 临时目录 `cache/autosave/` 存在
- 包含 `content_<uuid>.md` 和 `meta_<uuid>.properties`

**验证命令**:
```bash
adb shell run-as com.editor.nomadmark ls -la cache/autosave/
```

---

### 测试 2: 崩溃后恢复
**目标**: 验证应用崩溃后可以恢复内容

**步骤**:
1. 在编辑器中输入: "Recovery Test Content"
2. 等待自动保存（3秒 + 5秒最小间隔）
3. 强制停止应用: `adb shell am force-stop com.editor.nomadmark`
4. 重新启动应用
5. 观察是否显示恢复对话框

**预期结果**:
- 显示"发现未保存的内容"对话框
- 对话框显示文件名、时间、大小
- 点击"恢复"可以恢复内容

---

### 测试 3: 恢复对话框操作
**目标**: 测试恢复对话框的各种选项

**子测试 3.1: 点击"恢复"**
- 预期: 内容恢复到编辑器，临时文件删除

**子测试 3.2: 点击"放弃"**
- 预期: 临时文件删除，正常启动

**子测试 3.3: 点击"稍后"**
- 预期: 临时文件保留，正常启动，下次启动仍提示

---

### 测试 4: onPause 自动保存
**目标**: 验证应用进入后台时自动保存

**步骤**:
1. 在编辑器中输入: "Background Test"
2. 按 Home 键使应用进入后台
3. 检查临时文件是否更新

**预期结果**:
- 应用进入后台时触发自动保存
- 日志显示: "Activity paused, auto-saved changes"

---

### 测试 5: 过期文件清理
**目标**: 验证过期临时文件被清理

**步骤**:
1. 手动创建一个过期的临时文件（修改时间戳）
2. 启动应用
3. 检查过期文件是否被删除

**预期结果**:
- 7 天前的文件被清理
- 日志显示: "清理 X 个过期会话"

---

### 测试 6: 最小保存间隔
**目标**: 验证频繁编辑时不会过度保存

**步骤**:
1. 快速连续输入多个字符
2. 观察保存频率

**预期结果**:
- 至少间隔 5 秒才会保存
- 日志显示: "跳过频繁自动保存"

---

### 测试 7: 正常保存后清理临时文件
**目标**: 验证手动保存后临时文件被清理

**步骤**:
1. 编辑内容触发自动保存
2. 点击保存按钮
3. 检查临时文件是否被清理

**预期结果**:
- 保存成功后临时文件被删除
- SharedPreferences 标志被清除
