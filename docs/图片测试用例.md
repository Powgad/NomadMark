# 图片测试用例

## 测试目的

验证 Supernote A6 X2 Nomad 设备上的图片渲染功能是否正常工作。

## 测试环境

- **设备**: Supernote A6 X2 Nomad
- **分辨率**: 1872 × 1404 (300 DPI)
- **内存**: 4 GB RAM
- **显示**: E-ink 电子纸

## 测试文件位置

生成的 APK: `platforms/android/app/build/outputs/apk/debug/NomadMark-debug-1.0.0.apk`

## 测试用例

### 用例 1: 插入大尺寸图片

**步骤**:
1. 打开 NomadMark 应用
2. 点击底部工具栏的图片按钮
3. 选择"选择本地图片"
4. 从相册中选择一张大尺寸照片（> 4000px）
5. 等待图片处理完成
6. 切换到预览模式查看效果

**预期结果**:
- 图片插入成功，显示为 `![图片](file:///data/data/.../markdown_images/xxx.jpg)`
- 预览模式下图片显示正常
- 图片宽度不超过 1872px
- 无卡顿或闪退

### 用例 2: 插入多张图片

**步骤**:
1. 创建新文档
2. 连续插入 3-4 张不同尺寸的图片
3. 切换到预览模式

**预期结果**:
- 所有图片正常显示
- 内存占用稳定
- 无明显卡顿

### 用例 3: 图片持久化测试

**步骤**:
1. 插入图片到文档
2. 保存文档
3. 关闭应用
4. 重新打开文档

**预期结果**:
- 图片正常显示
- Markdown 内容保持完整

### 用例 4: 图片尺寸验证

**步骤**:
1. 插入大尺寸图片（如 4000×3000）
2. 检查生成的文件

**预期结果**:
- 图片保存在 `/data/data/com.editor.nomadmark/files/markdown_images/` 目录
- 文件格式为 JPEG
- 文件大小合理（< 500KB）

### 用例 5: 保存/重新加载测试

**步骤**:
1. 创建包含图片的 Markdown 文档
2. 保存文档
3. 关闭应用
4. 重新打开文档

**预期结果**:
- 图片正常加载
- 显示效果一致

## 内存验证

使用 Android Studio Profiler 或 ADB 命令监控内存：

```bash
# 查看内存占用
adb shell dumpsys meminfo com.editor.nomadmark
```

**预期内存占用**:
- 单张大图处理后: < 10 MB
- 4 张图片同时显示: < 30 MB
- 应用总内存: < 150 MB

## 故障排查

### 问题: 图片显示模糊

**可能原因**:
- 采样率过高
- JPEG 压缩质量过低

**解决方案**:
- 调整 `ImageProcessor.MAX_SCREEN_WIDTH`
- 提高 `ImageProcessor.JPEG_QUALITY`

### 问题: 插入图片后应用闪退

**可能原因**:
- OOM (内存不足)
- 图片过大

**解决方案**:
- 检查图片原始尺寸
- 调整采样率计算逻辑
- 使用 RGB565 格式

### 问题: 图片加载缓慢

**可能原因**:
- 磁盘 I/O 慢
- 解码性能问题

**解决方案**:
- 检查存储性能
- 使用硬件解码（如果可用）
