# 网络图片显示测试步骤

## 方法 1：使用图片测试 Activity

### 启动测试
```bash
# 连接设备
adb devices

# 启动图片测试 Activity
adb shell am start -n com.editor.nomadmark/.ImageDisplayTestActivity
```

### 在测试 Activity 中
1. 点击"测试网络图片"按钮
2. 观察 Markdown 渲染结果

**预期结果**：
- 如果看到 placeholder 图片（灰色方块，白色文字），说明 Markwon 网络图片功能正常
- 如果没有显示，说明 Markwon 图片配置有问题

---

## 方法 2：手动在 NomadMark 中测试

### 步骤
1. 打开 NomadMark 应用
2. 点击底部工具栏的图片按钮
3. 选择"输入网络图片链接"
4. 输入测试 URL：
   ```
   https://via.placeholder.com/300
   ```
5. 点击"插入"
6. 切换到预览模式（或使用分屏模式）

**预期结果**：
- 预览中显示 300x300 的灰色方块图片

---

## 方法 3：创建测试 Markdown 文件

### 创建测试文件
创建一个名为 `test_images.md` 的文件，内容如下：

```markdown
# 图片测试

## 网络图片测试 1

![Placeholder 300](https://via.placeholder.com/300)

## 网络图片测试 2

![Placeholder 500](https://via.placeholder.com/500)

## 测试完成

如果以上图片能显示，说明 Markwon 的网络图片功能正常。
```

### 将文件放到设备
```bash
# 推送到设备
adb push test_images.md /sdcard/Documents/

# 或者使用其他方式将文件放到设备的 Documents 目录
```

### 在 NomadMark 中打开
1. 打开 NomadMark
2. 点击"打开文件"
3. 选择 `test_images.md`

---

## 方法 4：直接在应用中创建测试内容

### 步骤
1. 打开 NomadMark
2. 创建新文档或编辑现有文档
3. 输入以下内容：

```markdown
# 网络图片测试

![测试图片](https://via.placeholder.com/300)
```

4. 切换到预览模式

---

## 检查结果

### 成功
- 看到灰色方块图片（placeholder）
- 图片上可能有 "300x300" 或 "Placeholder" 文字

### 失败
- 没有显示任何内容
- 只显示 Markdown 文本
- 显示图片占位符（alt 文本）

---

## 如果测试失败

### 检查网络连接
```bash
# 测试设备网络
adb shell ping -c 3 via.placeholder.com
```

### 查看日志
```bash
# 查看 Markwon 和图片相关日志
adb logcat -s Markwon:D ImagesPlugin:D
```

### 可能的问题
1. **网络权限** - 检查 AndroidManifest.xml 是否有网络权限
2. **Markwon 配置** - ImagesPlugin 可能需要额外配置
3. **Glide/Coil 配置** - 图片加载库可能未正确初始化

---

## 快速验证命令

```bash
# 一键测试网络图片
adb shell am start -n com.editor.nomadmark/.ImageDisplayTestActivity && sleep 2 && adb logcat -s ImageDisplayTest:D
```

这将启动测试 Activity 并显示相关日志。
