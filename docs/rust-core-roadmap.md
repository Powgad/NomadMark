# Rust Core 开发路线图

## 项目概述

**目标**: 将 NomadMark 的渲染引擎从 Markwon (Java) 迁移到 Rust Core，提供更好的性能、内存控制和墨水屏优化。

**当前状态**: 实验性阶段，基础渲染功能可用，缺少大部分 Markdown 扩展语法支持。

---

## 依赖库评估

### 数学公式渲染

| 库 | 优势 | 劣势 | 推荐度 |
|----|------|------|--------|
| `katex-rs` | 功能完整 | 编译复杂 | ⭐⭐⭐ |
| `tex2mathml` | 轻量 | 输出 HTML 需转换 | ⭐⭐⭐⭐ |
| 自实现简化版 | 完全控制 | 功能有限 | ⭐⭐⭐⭐⭐ |

**推荐**: 使用 `tex2mathml` + 自定义渲染器，仅支持常用公式

### 图片加载

| 方案 | 优势 | 劣势 |
|------|------|------|
| JNI 回调 Kotlin | 复用 Android API | 跨边界开销 |
| Rust `image` crate | 完全原生 | 格式支持有限 |
| 混合方案 | 灵活性高 | 复杂度增加 |

**推荐**: JNI 回调 Kotlin 端，使用 Glide/Coil 加载

## 架构设计

### 渲染管道

```
Markdown 文本
    ↓
StreamingParser (解析)
    ↓
AST (BlockNode + InlineNode)
    ↓
Layouter (布局计算)
    ↓
RenderCommand[] (渲染命令)
    ↓
RenderCommandExecutor (Kotlin)
    ↓
Canvas 绘制
```

### 内存管理

```rust
// 内存预算
const TOTAL_BUDGET: usize = 100 * 1024 * 1024; // 100MB

const GLYPH_CACHE_QUOTA: usize = 20 * 1024 * 1024;   // 20MB
const RENDER_CACHE_QUOTA: usize = 60 * 1024 * 1024;   // 60MB
const PARSER_QUOTA: usize = 20 * 1024 * 1024;         // 20MB
```

### JNI 边界优化

```rust
// 零拷贝传输
pub struct RenderResult {
    // 使用 Box::leak 返回稳定指针
    pub commands: *const RenderCommand,
    pub count: usize,
    // DirectByteBuffer 用于脏矩形
    pub dirty_rects: *const i32,
    pub dirty_count: usize,
}
```
