# 图片功能调试指南

## 问题：图片插入后不显示

如果您通过 NomadMark 插入了图片，但预览中没有显示，请按照以下步骤检查：

## 步骤 1：检查图片是否被复制到私有目录

### 方法 A：使用 ADB 命令检查

```bash
# 连接设备
adb devices

# 进入 shell
adb shell

# 切换到应用目录
cd /data/data/com.editor.nomadmark/files

# 检查图片目录
ls -la markdown_images/

# 应该看到 .jpg 文件
```

### 方法 B：使用图片测试 Activity

应用已包含一个图片测试 Activity，可以通过以下方式启动：

```bash
# 启动图片测试 Activity
adb shell am start -n com.editor.nomadmark/.ImageDisplayTestActivity
```

在测试 Activity 中：
1. 点击"测试网络图片" - 验证 Markwon 图片功能是否正常
2. 点击"测试本地图片" - 查看图片目录状态

## 步骤 2：检查 Logcat 日志

```bash
# 查看图片处理相关日志
adb logcat -s ImageProcessor:D MarkdownEditorActivity:D
```

插入图片时应该看到类似日志：

```

MarkdownEditorActivity: 接收到的 URI: content://...
MarkdownEditorActivity: 开始处理图片...
ImageProcessor: 开始处理图片: content://...
ImageProcessor: 原始尺寸: 4000×3000, 内存: 45.8MB
ImageProcessor: 采样率 inSampleSize: 4
ImageProcessor: 采样后尺寸: 1000×750, 内存: 1.4MB
ImageProcessor: 保存图片: xxx.jpg, 大小: 125KB
MarkdownEditorActivity: 图片处理结果: markdown_images/xxx.jpg
MarkdownEditorActivity: 插入图片 URL: file:///data/data/.../markdown_images/xxx.jpg
MarkdownEditorActivity: ========== 插入图片结束 ==========
```

## 步骤 3：检查 Markdown 内容

查看编辑器中生成的 Markdown 格式：

**正确格式**：
```markdown
![图片](file:///data/data/com.editor.nomadmark/files/markdown_images/xxx.jpg)
```

**错误格式示例**：
```markdown
![图片](content://...)  // 不支持 content:// 协议
```

## 步骤 4：测试网络图片

1. 点击图片按钮
2. 选择"输入网络图片链接"
3. 输入: `https://via.placeholder.com/300`
4. 切换到预览模式

**如果网络图片能显示**：
- 说明 Markwon 图片功能正常
- 问题可能出在本地图片处理

**如果网络图片也不显示**：
- 可能是 Markwon 配置问题
- 检查 Coil 初始化是否成功

## 步骤 5：检查 Coil 初始化

```bash
# 查看 Coil 初始化日志
adb logcat -s SupernoteImageLoader:D
```

应该看到：
```
SupernoteImageLoader: 图片加载器已初始化 - 目标尺寸: 1872
```

## 常见问题及解决方案

### 问题 1：图片目录为空

**可能原因**：
- ImageProcessor.processAndSaveImage() 返回 null
- 权限问题
- 图片格式不支持

**解决方案**：
查看 logcat 日志中的错误信息

### 问题 2：Markdown 格式不正确

**可能原因**：
- insertImage() 方法生成错误的格式

**解决方案**：
手动编辑 Markdown，确保格式为 `![图片](file:///...)`

### 问题 3：图片存在但不显示

**可能原因**：
- Markwon 不支持 file:// 协议
- Coil 配置问题

**解决方案**：
- 检查 Markwon 是否正确配置了 Coil
- 尝试使用绝对路径而非 file:// 协议

### 问题 4：OOM (内存不足)

**可能原因**：
- 图片过大
- 采样率计算错误

**解决方案**：
- 检查日志中的内存占用
- 调整 MAX_SCREEN_WIDTH 常量

## 手动验证测试

### 创建测试文件

创建一个包含以下内容的 Markdown 文件：

```markdown
# 图片测试

## 网络图片测试
![网络图片](https://via.placeholder.com/300)

## 本地图片测试
![本地图片](file:///data/data/com.editor.nomadmark/files/markdown_images/test.jpg)
```

### 预期结果

- 网络图片应该正常显示
- 本地图片如果文件存在，也应该显示

## 性能监控

### 监控内存占用

```bash
# 查看应用内存占用
adb shell dumpsys meminfo com.editor.nomadmark
```

### 监控图片加载性能

```bash
# 查看 Coil 相关日志
adb logcat -s Coil:D
```

## 调试工具

### 使用 ImageTestUtils

代码中包含 `ImageTestUtils` 类，可以用于调试：

```kotlin
// 检查图片目录
val info = ImageTestUtils.checkImageDirectory(context)
Log.d("Test", info)

// 创建测试图片
val path = ImageTestUtils.createTestImage(context)

// 验证图片文件
val exists = ImageTestUtils.verifyImageFile(context, "markdown_images/test.jpg")
```
