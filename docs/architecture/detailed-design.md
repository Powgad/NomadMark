手动保存：

<img src="docs/assets/designs/Mark down编辑器竖屏页面布局_Page_23.png" title="" alt="Mark down编辑器竖屏页面布局Page23" style="zoom:25%;">

<img src="docs/assets/designs/Mark down编辑器竖屏页面布局_Page_22.png" title="" alt="Mark down编辑器竖屏页面布局Page22" style="zoom:25%;">

关机开机或重启后再回到此界面也会弹出该提示框，防止文件丢失。

## 5. 编辑器中可能出现的冲突和规避方法

### 5.1 渲染显示冲突

#### 5.1.1 语法高亮冲突

| 冲突表现     | 原因               | 规避方法          |
| -------- | ---------------- | ------------- |
| 高亮颜色无法区分 | 墨水屏灰阶有限，彩色语法高亮失效 | 提供"单色模式"选项    |
| 背景色块产生残影 | 代码块深色背景在刷新时留下痕迹  | 使用黑白点组合成灰色背景块 |

#### 5.1.2 预览/编辑模式切换冲突

| 冲突表现   | 原因            | 规避方法                              |
| ------ | ------------- | --------------------------------- |
| 光标位置丢失 | 切换后无法回到原位置    | 记住`EditText.selectionStart`，切换后恢复 |
| 渲染不一致  | 编辑模式和预览模式显示差异 | 在预览模式下高亮当前编辑段落                    |

### 5.2 输入编辑冲突

#### 5.2.1 光标渲染冲突

| 冲突表现    | 原因             | 规避方法                                  |
| ------- | -------------- | ------------------------------------- |
| 光标闪烁不可见 | 墨水屏刷新率低，光标闪烁被吞 | 使用"块光标"(Block Cursor)替代闪烁光标，用反转色块显示位置 |
| 光标移动延迟  | 光标跟随触摸位置有延迟    | 实现"光标预判"，触摸时立即显示预测位置，真实位置延迟更新         |
| 光标跳动    | 软键盘弹出导致光标位置重算  | 锁定光标行号，键盘弹出后恢复到原行                     |

#### 5.2.2 文本选择冲突

| 冲突表现    | 原因         | 规避方法                                              |
| ------- | ---------- | ------------------------------------------------- |
| 长按选择延迟  | 墨水屏长按响应慢   | 降低`ViewConfiguration.getLongPressTimeout()`至300ms |
| 多段落选择卡顿 | 大文本选择时频繁刷新 | 使用"虚拟选择"：仅在松手时真正高亮选中区域                            |

#### 5.2.3 手写笔输入冲突

| 冲突表现    | 原因          | 规避方法                 |
| ------- | ----------- | -------------------- |
| 笔迹与光标冲突 | 手写输入时光标位置跳动 | 手写模式下隐藏光标，显示"手写区域"边框 |
| 手掌误触    | 手掌支撑屏幕触发滚动  | 仅响应笔尖输入              |
| 墨迹残留    | 手写笔迹清除不全    | 手写结束时立即触发局部刷新清除轨迹    |

## 7. 使用场景

场景 : Supernote 用户编辑 Markdown 文档

**参与者**：Supernote 用户前置条件：用户已登录 Supernote 设备，已打开 Markdown编辑器

编辑器使用流程：故事性

1. 小王新建了Markdown 文件，看到了新手引导
2. 小王进入到编辑模式，通过外接键盘/软键盘输入文本
3. 小王可以开启修订功能，使用手写笔划特定手势进行修订
4. 小王也可以开启分屏模式，同时看到编辑区和预览区展现内容，双区均可进行内容修改
5. 小王开启预览模式，开启全屏预览，查看渲染效果（预览模式下不支持光标编辑，可以使用修订功能）
6. 小王可以点击保存按键进行文件保存
7. 小王在中途离开界面会跳出提醒保存弹窗防止文件丢失
8. 如果设备断电，重启后也会跳出保存提醒防丢失

**异常流程**：

* 如果在编辑过程中跳转到文件管理并删除文件，系统会提示小王是否确认删除文件

* 如果编辑过程中应用崩溃，系统应会在重启后恢复

## 9. 语法优先级和常见冲突场景

### 9.1 优先级排序（从高到低）

    1. HTML 标签 (最高优先级)
    2. 代码块 / 行内代码
    3. 数学公式
    4. 链接 / 图片
    5. 强调（粗体/斜体）
    6. 删除线
    7. 标题
    8. 列表
    9. 引用
    10. 分隔线
    11. 普通文本 (最低优先级)

### 9.2 常见冲突场景

#### ９.２.１粗体与斜体嵌套

**输入示例：**
    ***粗斜体文本***
    **_粗斜体文本_**
    _*斜体中的粗体_*

**优先级规则：** 内层语法优先解析

**解析结果：**

* `***文本***` → 粗斜体（三者都生效）
* `**_文本_**` → 粗体（下划线被忽略，`_` 作为普通字符）
* `_*文本_*` → 斜体（星号被忽略，`*` 作为普通字符）

#### ９.２.２代码与强调冲突

**输入示例：**
    `**代码中的星号**`
    **`粗体中的代码`**

**优先级规则：** 代码标记 `` ` `` 优先级高于强调符号

**解析结果：**

* `` `**文本**` `` → 行内代码（星号不触发粗体）
* `**`代码`**` → 粗体（反引号作为普通字符）

* * *

#### ９.２.３链接与强调冲突

**输入示例：**
    **[粗体中的链接](https://example.com)**
    [*斜体链接*](https://example.com)

**优先级规则：** 链接解析优先于强调

**解析结果：**

* `**[链接](url)**` → 粗体（链接生效后再应用粗体）
* `[*文本*](url)` → 链接（星号不触发斜体）
  
  

#### ９.２.４标题与列表冲突

**输入示例：**
    # 标题中的 - 列表符号
    - 列表中的 # 符号

**优先级规则：** 行首解析符号决定语法类型

**解析结果：**

* `# 标题中的 -` → 一级标题（`-` 作为普通字符）
* `- 列表中的 #` → 无序列表（`#` 作为普通字符）

* * *

#### ９.２.５引用与代码块冲突

**输入示例：**
    > ```python
    > code here
    > ```

**优先级规则：** 引用标记 `>` 嵌套代码块

**解析结果：**

* 引用内的代码块（两者都生效）

**反向示例：**
    ```python
    > quoted code
    **解析结果：**
    - 代码块优先（`>` 作为代码内容）

## 10. Android 平台层级架构设计（2026-07-06 更新）

### 10.1 独立层级架构

为解决光标显示和模式切换问题，Android 平台采用独立的层级架构设计。

#### 层级结构

```
RelativeLayout (根容器)
├── 顶部工具栏 (toolbar_top)
├── 搜索栏 (search_bar)
├── 层1：编辑层 (editor_layer)
│   └── ScrollView → EditText (支持光标、键盘编辑)
├── 层2：预览层 (preview_layer)
│   └── ScrollView → TextView (只读，无光标)
├── 层3：分屏层 (split_layer)
│   ├── 上半区：预览区 (ObservableScrollView → TextView)
│   └── 下半区：编辑区 (ObservableScrollView → EditText)
├── 层4：手势层 (gesture_layer)
│   └── GestureOverlayView (修订模式启用手势识别)
├── 底部快捷栏 (toolbar_bottom)
├── 目录面板 (toc_panel)
└── 键盘标识 (keyboard_indicator)
```

#### 层级特性

| 层级 | 主要组件 | 光标 | 键盘编辑 | 修订模式 |
|------|----------|------|----------|----------|
| 编辑层 | EditText | ✅ 默认显示 | ✅ 支持 | ✅ 手势修订 |
| 预览层 | TextView | ❌ 无 | ❌ 不支持 | ✅ 手势修订 |
| 分屏层-预览 | TextView | ❌ 无 | ❌ 不支持 | ✅ 手势修订 |
| 分屏层-编辑 | EditText | ✅ 显示 | ✅ 支持 | ✅ 手势修订 |
| 手势层 | GestureOverlayView | - | - | ✅ 启用时覆盖 |

### 10.2 显示模式管理

#### DisplayMode 枚举

```kotlin
enum class DisplayMode {
    EDIT,    // 编辑模式
    PREVIEW, // 预览模式
    SPLIT    // 分屏模式
}
```

#### 模式切换逻辑

**F11 键切换**：在 EDIT ↔ PREVIEW ↔ SPLIT 之间循环切换

**修订按钮**：在当前模式下开启/关闭修订功能

**光标控制规则**：
- 编辑模式：光标默认显示，开启修订后消失
- 预览模式：无光标，开启修订后支持手势
- 分屏模式：编辑区有光标，开启修订后上下两区都支持手势

### 10.3 手势层设计

#### 覆盖层属性

```xml
<com.editor.nomadmark.GestureOverlayView
    android:id="@+id/gesture_layer"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:visibility="gone"
    android:clickable="false"
    android:focusable="false" />
```

#### 区域判断（分屏模式）

根据触摸 Y 坐标判断操作区域：
- **上半区**（预览区）：操作同步到编辑内容
- **下半区**（编辑区）：直接应用操作

```kotlin
val splitRatio = 0.6f  // 预览区占 60%
val dividerY = splitLayer.top + splitHeight * splitRatio

if (touchY < dividerY) {
    // 上半区（预览区）操作
} else {
    // 下半区（编辑区）操作
}
```

### 10.4 数据同步机制

#### 编辑器同步

- editorText ↔ splitEditorText 双向同步
- 通过 `syncEditors()` 函数实现
- 使用 `isSyncing` 标志防止循环同步

#### 预览更新

- 监听文本变化（`textWatcher`）
- 自动调用 `updatePreview()` 更新预览
- 支持预览层和分屏层的预览区
