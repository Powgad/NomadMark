# NomadMark 双重渲染架构文档

## 概述

NomadMark 采用双重渲染架构，以兼顾编辑性能和预览效果。

## 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                     NomadMark 应用层                          │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌──────────────────┐         ┌──────────────────┐         │
│  │   编辑模式        │         │   预览模式        │         │
│  │                  │         │                  │         │
│  │  MarkdownEditor  │         │  PreviewView     │         │
│  │  (Rust Core)     │         │  (Markwon)       │         │
│  │                  │         │                  │         │
│  │  • 原始文本       │         │  • 渲染效果       │         │
│  │  • 快速输入       │         │  • 数学公式       │         │
│  │  • 光标管理       │         │  • 表格渲染       │         │
│  │  • 低延迟         │         │  • 完整样式       │         │
│  └──────────────────┘         └──────────────────┘         │
│                                                               │
│  ┌──────────────────────────────────────────────┐           │
│  │           分屏模式 (Split Mode)               │           │
│  ├──────────────────────────────────────────────┤           │
│  │  上方: Markwon 预览区                        │           │
│  │  ────────────────────────────────────────   │           │
│  │  下方: Rust Core 编辑区                      │           │
│  └──────────────────────────────────────────────┘           │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

## 职责划分

| 组件 | 技术栈 | 用途 | 优势 |
|------|--------|------|------|
| **Rust Core** | Rust + JNI | 编辑模式 | • 高性能<br>• 低延迟<br>• 内存效率高<br>• 完全可控 |
| **Markwon** | Java/Kotlin | 预览模式 | • 功能完整<br>• 成熟稳定<br>• 数学公式支持<br>• 社区支持 |

## 使用场景

### 1. 编辑模式
```
触发: 点击编辑按钮、默认模式
显示: markdown_editor_view (Rust Core)
内容: 原始 Markdown 文本
```

### 2. 预览模式
```
触发: 点击预览按钮
显示: previewText (Markwon 渲染)
内容: 渲染后的 Markdown
```

### 3. 分屏模式
```
触发: 点击分屏按钮
显示: 
  - 上方: splitPreviewText (Markwon)
  - 下方: markdown_editor_view (Rust Core)
```

## 语法兼容性

两套渲染系统支持相同的 Markdown 语法：

| 语法 | Rust Core | Markwon | 状态 |
|------|-----------|---------|------|
| 标题 (H1-H6) | ✅ | ✅ | ✅ 一致 |
| 粗体 | ✅ | ✅ | ✅ 一致 |
| 斜体 | ✅ | ✅ | ✅ 一致 |
| 删除线 | ✅ | ✅ | ✅ 一致 |
| 代码块 | ✅ | ✅ | ✅ 一致 |
| 表格 | ✅ | ✅ | ✅ 一致 |
| 任务列表 | ✅ | ✅ | ✅ 一致 |
| 数学公式 | 🔄 | ✅ | 🔄 Rust Core 开发中 |
| 引用块 | 🔄 | ✅ | 🔄 Rust Core 开发中 |

## 冲突避免原则

### 1. 明确使用边界
- **编辑时**：只显示 Rust Core 渲染
- **预览时**：只显示 Markwon 渲染
- **不同时渲染**：避免两个系统同时渲染同一内容

### 2. 数据流单向
```
用户输入 → Rust Core (编辑器) → 文本内容 → Markwon (预览)
                ↑                                      ↓
                └──────────── 只同步文本 ────────────────┘
```

### 3. 渲染状态隔离
- 编辑状态由 Rust Core 管理
- 预览状态由 Markwon 管理
- 通过文本内容进行状态同步

## 开发注意事项

1. **添加新功能时**：确保两边都支持或明确只在一侧支持
2. **测试时**：分别测试编辑和预览效果
3. **性能优化时**：优先优化 Rust Core（编辑体验是核心）
4. **样式调整时**：确保两边渲染效果视觉一致
