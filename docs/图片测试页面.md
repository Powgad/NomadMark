# 图片测试页面

## 测试图片显示

如果您已经通过 NomadMark 插入了图片，但预览中没有显示，请检查以下几点：

### 1. 检查图片目录

使用 ADB 命令检查图片是否被复制到私有目录：

```bash
adb shell
su
cd /data/data/com.editor.nomadmark/files
ls -la markdown_images/
```

### 2. 检查 Markdown 格式

正确的图片格式应该是：

```markdown
![图片](file:///data/data/com.editor.nomadmark/files/markdown_images/xxx.jpg)
```

### 3. 可能的问题和解决方案

#### 问题 A: 图片目录为空

**原因**: ImageProcessor.processAndSaveImage() 返回了 null

**解决方案**: 检查 logcat 日志，查看图片处理过程中的错误信息

#### 问题 B: Markdown 格式不正确

**原因**: insertImage() 方法可能生成了错误的格式

**解决方案**: 手动编辑 Markdown，确保格式正确

#### 问题 C: Markwon 无法解析 file:// 协议

**原因**: Markwon 4.6.2 的 ImagesPlugin 可能不支持 file:// 协议

**解决方案**: 需要配置 Markwon 使用 Coil 加载本地图片

### 4. 手动测试

创建一个简单的测试文件：

```markdown
# 图片测试

![测试图片](file:///data/data/com.editor.nomadmark/files/markdown_images/test.jpg)

## 网络图片测试

![网络图片](https://via.placeholder.com/150)
```

### 5. 检查 logcat 日志

运行应用并插入图片后，查看 logcat 日志：

```bash
adb logcat -s ImageProcessor:D MarkdownEditorActivity:D
```

应该看到类似的日志：

```
ImageProcessor: 开始处理图片: content://...
ImageProcessor: 原始尺寸: 4000×3000, 内存: 45.8MB
ImageProcessor: 采样率 inSampleSize: 4
ImageProcessor: 采样后尺寸: 1000×750, 内存: 1.4MB
ImageProcessor: 保存图片: xxx.jpg, 大小: 125KB
MarkdownEditorActivity: 插入图片 URL: file:///data/data/.../markdown_images/xxx.jpg
```

### 6. 确认 Coil 初始化

检查 Coil 是否正确初始化：

```bash
adb logcat -s SupernoteImageLoader:D NomadMarkApplication:D
```

应该看到：

```
SupernoteImageLoader: 图片加载器已初始化 - 目标尺寸: 1872
```

### 7. 测试网络图片

尝试插入网络图片来验证 Markwon 的图片功能是否正常：

1. 点击图片按钮
2. 选择"输入网络图片链接"
3. 输入: `https://via.placeholder.com/300`

如果网络图片能显示，说明 Markwon 的图片功能正常，问题可能出在本地图片的处理上。
