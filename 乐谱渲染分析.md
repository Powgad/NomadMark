# 代码块渲染和乐谱渲染的关系分析

## 1. 代码块边框 (`applyCodeBlockBorder`)

### 检测流程
1. 检测 `CodeBlockSpan`（Markwon 生成的代码块 Span）
2. 检查是否包含 `ReplacementSpan`（图片等）
3. 如果不包含，为代码块的每一行应用边框

### 关键代码
```kotlin
val codeSpans = spannable.getSpans(0, spanned.length, codeBlockSpanClass)
for (span in codeSpans) {
    val start = spannable.getSpanStart(span)
    val end = spannable.getSpanEnd(span)
    val replacementSpans = spannable.getSpans(start, end, ReplacementSpan::class.java)
    if (replacementSpans.isNotEmpty()) {
        continue  // 跳过包含 ReplacementSpan 的代码块
    }
    applyBorderToLines(spannable, start, end, flags)
}
```

## 2. 乐谱渲染 (`applyMusicSheetRendering`)

### 检测流程
1. 检测 `CodeBlockSpan`
2. 解析文本内容，检查是否是乐谱块（```music 或 ```简谱）
3. 如果是乐谱块，应用 `MusicSheetSpan`（一个 `ReplacementSpan`）

### 关键代码
```kotlin
val codeSpans = spannable.getSpans(0, spanned.length, codeBlockSpanClass)
for (span in codeSpans) {
    val start = spannable.getSpanStart(span)
    val end = spannable.getSpanEnd(span)
    val blockText = spanned.subSequence(start, end).toString()
    val musicData = parseMusicBlock(blockText)
    if (musicData != null) {
        spannable.setSpan(musicSpan, start, end, Spanned.SPAN_EXCLUSIVE_EXCLUSIVE)
    }
}
```

## 3. 两者的关系

### 共同点
- 都基于 `CodeBlockSpan` 检测
- 都处理相同类型的代码块

### 冲突点
1. **代码块边框** 为每行应用边框
2. **乐谱渲染** 应用一个 `ReplacementSpan` 来替换整个块

### ReplacementSpan 的行为
- `ReplacementSpan` 会替换文本内容，调用 `draw()` 方法来渲染
- 但 `ReplacementSpan` **不会自动移除其他 Span**
- 如果 `CodeBlockSpan` 仍然存在，它可能会影响显示

## 4. 问题分析

### 当前问题
乐谱渲染后和普通代码块效果相同

### 可能原因
1. **CodeBlockSpan 仍然可见**
   - `MusicSheetSpan` 是 `ReplacementSpan`，它替换文本内容
   - 但 `CodeBlockSpan` 可能仍然存在并影响显示

2. **调用顺序问题**
   - 如果先应用代码块边框，后应用乐谱渲染
   - 边框可能会覆盖 `MusicSheetSpan`

3. **ReplacementSpan 的 `getSize()` 方法**
   - 如果返回的高度不正确，`draw()` 方法可能不会被调用
   - 或者高度太小，无法看到内容

## 5. 解决方案

### 方案 1：移除 CodeBlockSpan
在应用 `MusicSheetSpan` 之前，移除原始的 `CodeBlockSpan`

### 方案 2：确保正确的高度
确保 `MusicSheetSpan.getSize()` 返回足够的高度

### 方案 3：正确的调用顺序
先应用乐谱渲染，再应用代码块边框，这样边框会跳过乐谱块
