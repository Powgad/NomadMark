# NomadMark 渲染测试集

本文档用于验证 Rust Core 渲染功能的正确性和完整性。

## 测试清单

使用此文档测试所有 Markdown 语法元素的渲染效果。

---

## 一、基础语法

### 1.1 标题 (Headings)

```markdown
# 一级标题 (H1)

## 二级标题 (H2)

### 三级标题 (H3)

#### 四级标题 (H4)

##### 五级标题 (H5)

###### 六级标题 (H6)
```

**预期渲染**：
- H1: 24pt, 粗体
- H2: 20pt, 粗体
- H3: 18pt, 粗体
- H4: 16pt, 粗体
- H5: 14pt, 粗体
- H6: 12pt, 粗体

---

### 1.2 强调 (Emphasis)

```markdown
**粗体文本** (Strong)
*斜体文本* (Emphasis)
***粗斜体*** (Strong + Emphasis)
~~删除线~~ (Strikethrough)
```

**预期渲染**：
- 粗体：700 字重
- 斜体：倾斜字体
- 删除线：中间删除线

---

### 1.3 行内代码 (Inline Code)

```markdown
这是 `行内代码` 示例。
```

**预期渲染**：
- 等宽字体
- 浅灰背景 (#F5F5F5)
- 左右各 4px 内边距

---

## 二、块级元素

### 2.1 代码块 (Code Blocks)

````markdown
```python
def hello_world():
    print("Hello, World!")
```
````

**预期渲染**：
- 语法高亮（如果指定语言）
- 浅灰背景
- 等宽字体
- 左右内边距 12px

---

### 2.2 引用块 (Blockquote)

```markdown
> 这是一级引用
>
> > 这是嵌套引用
```

**预期渲染**：
- 左边框：2px 实线，颜色 #CCCCCC
- 背景色：#F5F5F5
- 左内边距：12px
- 嵌套层级增加边框间距

---

### 2.3 列表 (Lists)

#### 无序列表

```markdown
- 第一项
- 第二项
  - 嵌套项
- 第三项
```

#### 有序列表

```markdown
1. 第一项
2. 第二项
   1. 嵌套项
3. 第三项
```

#### 任务列表

```markdown
- [ ] 未完成任务
- [x] 已完成任务
```

**预期渲染**：
- 列表标记正确显示
- 嵌套缩进一致
- 任务列表复选框可点击（编辑模式）

---

### 2.4 分割线 (Thematic Break)

```markdown
---
或
***
或
___
```

**预期渲染**：
- 1px 实线
- 颜色 #DDDDDD
- 上下边距各 16px

---

## 四、链接与图片

### 4.1 链接 (Links)

```markdown
[内联链接](https://example.com)
[带标题的链接](https://example.com "鼠标悬停显示")
```

**预期渲染**：
- 蓝色文字（或 E-ink 模式下正常颜色）
- 无下划线（E-ink 优化）

### 4.2 图片 (Images)

```markdown

```

**预期渲染**：
- 图片自适应宽度
- 保持宽高比
- 标题悬停显示（如果支持）

- 字号为正文的 1.2 倍

## 六、高级功能

### 6.1 下划线与高亮

```markdown
++下划线文本++ (Underline)
==高亮文本== (Highlight)
```

**预期渲染**：
- 下划线：下划线装饰
- 高亮：黄色背景 (#FFFF00) 或 E-ink 模式下灰色背景

---

### 6.2 上标与下标

```markdown
H~2~O (下标)
x^2^ (上标)
```

**预期渲染**：
- 下标：基线下方，字号 0.8 倍
- 上标：基线上方，字号 0.8 倍

---

### 6.3 脚注 (Footnotes)

```markdown
这里有脚注引用[^1]。

[^1]: 这是脚注的内容。
```

**预期渲染**：
- 引用显示为上标数字
- 脚注内容在文档末尾
- 可点击跳转

---

### 6.4 Callout

```markdown
:::info
这是一个信息提示框。
:::

:::warning
这是一个警告提示框。
:::

:::note
这是一个注释提示框。
:::
```

**预期渲染**：
- 左侧彩色条（info=蓝, warning=黄, note=绿）
- 浅色背景
- 图标 + 标题 + 内容

### 7.2 性能测试（大文件）

使用此内容生成一个大文件（>1MB）测试渲染性能：

```markdown
# 性能测试

<!-- 重复 1000 次 -->
## 第 N 节

这是一个测试段落，包含 **粗体** 和 *斜体* 文本。

- 列表项 1
- 列表项 2

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |

```

**验收标准**：
- 渲染时间 < 1 秒
- 内存占用 < 100MB
- 滚动流畅

## 八、E-ink 特定测试

### 8.1 对比度测试

```markdown
# 黑色文字 (应该清晰可见)

**粗体标题**

[链接文字]（应该在 E-ink 上可见）

`代码文本`（背景色应与正文有足够对比度）

> 引用块（边框应该清晰可见）
```

**验收标准**：
- 所有文字与背景对比度 > 4.5:1
- 在 E-ink 设备上清晰可读

---

### 8.2 刷新区域测试

修改单个字符，验证局部刷新：

```markdown
# 测试标题

这是一段文字，其中包含一个**粗体**单词。

修改：将"粗体"改为"斜体"，应该只刷新局部区域。
```

**验收标准**：
- 只刷新变化区域
- 无闪烁
- 无重影
